# 桌面声音采集说明

Meeting Recorder 支持把 **桌面声音** 作为语音来源。

## 前端使用方式

在页面顶部的 **麦克风** 下拉框中选择：

- `Default` / 具体麦克风设备：采集麦克风
- `桌面声音`：采集系统播放的声音

选择 `桌面声音` 后，浏览器会弹出共享窗口。你需要：

1. 选择要共享的屏幕、窗口或标签页
2. **勾选“共享音频”**（或类似选项）
3. 确认共享

如果没有勾选共享音频，系统通常只能共享画面，无法拿到桌面音频轨道。

---

## Windows 11 常见配置方法

### 方法一：浏览器共享屏幕时勾选“共享音频”

这是最简单的方法，推荐优先使用。

适用步骤：

1. 在 Meeting Recorder 中选择 `桌面声音`
2. 浏览器弹出“共享屏幕/窗口/标签页”对话框
3. 勾选 **共享音频** / **Share system audio**
4. 再点击共享

适合：

- Chrome
- Edge

注意：

- 有些浏览器只对“标签页音频”支持更好
- 某些情况下共享整个桌面时音频支持不稳定
- 如果主机扬声器被静音，某些浏览器 / 系统组合下可能导致桌面音频轨道没有有效信号
- 如果切到了 Virtual Audio Cable，请确认播放器声音确实输出到了该虚拟设备，而不是仍然走默认扬声器

---

### 方法二：启用 Windows 的“立体声混音”

如果浏览器共享音频不稳定，可以尝试系统级录制设备。

操作路径：

1. 打开 **设置 → 系统 → 声音**
2. 找到 **更多声音设置**（旧控制面板入口）
3. 进入 **录制** 标签页
4. 找到 **立体声混音（Stereo Mix）**
5. 右键启用
6. 在浏览器的麦克风设备列表里选择它（如果能显示）

说明：

- 不同声卡驱动不一定提供“立体声混音”
- 某些笔记本默认隐藏，需要右键显示禁用设备

---

### 方法三：使用虚拟音频线缆

如果系统本身没有立体声混音，可以使用虚拟音频设备。

常见工具：

- VB-Audio Virtual Cable
- VoiceMeeter
- 其他虚拟声卡软件

思路：

1. 把系统输出改到虚拟声卡
2. 让播放器、会议软件或浏览器标签页的声音真的输出到该虚拟设备
3. 在 Meeting Recorder 中选择合适的采集方式：
   - 如果浏览器共享桌面音频稳定，就直接选 `桌面声音`
   - 如果你要录虚拟声卡本身的输入，可改选对应录音设备
4. 软件即可采集系统声音

Virtual Audio Cable 排查建议：

- 检查 **Windows 输出设备** 是否真的切到 Virtual Cable
- 检查 **具体播放器/会议软件** 有没有单独指定输出设备
- 检查浏览器共享时是否采集的是当前系统输出，而不是旧设备
- 如果看起来“在播声音但没有波形”，尝试重新选择共享目标并重新勾选共享音频

适合：

- 需要稳定录系统音
- 要同时录会议软件、播放器、浏览器声音

---

## 浏览器支持情况

### Chrome / Edge

- 对 `getDisplayMedia({ video: true, audio: true })` 支持最好
- 一般可直接共享标签页音频
- 桌面音频支持通常优于 Firefox

### Firefox

- 对桌面音频支持相对弱一些
- 某些平台下可能只能共享画面，没有音频
- 更容易遇到必须额外配置的问题

### Safari

- 桌面音频采集支持较弱
- 不建议作为主要方案

---

## 当前实现说明

Web 端使用浏览器的 `getDisplayMedia` 获取桌面音频：

```javascript
navigator.mediaDevices.getDisplayMedia({
  video: true,
  audio: true,
})
```

说明：

- 请求桌面音频时通常必须同时请求视频轨道
- 实际转录只使用音频轨道
- 如果用户停止共享，录音会自动结束

## 常见问题

### 1. 选择了“桌面声音”，但没有声音

常见原因：

- 没勾选“共享音频”
- 浏览器不支持当前共享方式的音频
- 共享的是窗口/屏幕，但该模式不提供音频轨道
- 主机扬声器被静音，导致某些浏览器 / 系统组合下没有有效输出信号
- 系统声音已经切到 Virtual Cable，但浏览器共享的并不是那个输出路径

建议：

- 优先用 Chrome 或 Edge
- 优先测试“共享标签页并勾选共享音频”
- 确认视频/播放器真的在出声，而不是仅仅时间轴在走
- 如果使用 Virtual Cable，确认播放器输出设备与 Windows 当前输出设备一致
- 如果共享后依然没声音，取消共享后重新选择一次，并重新勾选共享音频

### 2. 只有画面，没有音频轨道

这是浏览器/系统组合限制，建议：

- 改用 Chrome/Edge
- 改用“立体声混音”
- 改用 VB-Audio Virtual Cable

### 3. 录制过程中共享结束了

当用户手动结束屏幕共享后，桌面声音输入会自动中断，页面会停止当前录音。

### 4. 已切到 Virtual Cable，但仍然没音频

建议按这个顺序检查：

1. Windows 的默认输出设备是否真的是 Virtual Cable
2. 播放器 / 会议软件是否单独指定了别的输出设备
3. 浏览器共享时是否重新勾选了共享音频
4. Meeting Recorder 顶部是否选择了 `桌面声音`
5. 如果仍无效，尝试不用 `桌面声音`，而是直接选择 Virtual Cable 对应输入设备
